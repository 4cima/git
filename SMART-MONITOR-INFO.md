---

## 🎯 الفرق عن المراقب العادي

### ❌ المراقب العادي (تم إيقافه):
- إعادة تشغيل عمياء
- لا يحلل المشاكل
- قد يدخل في دوامة

### ✅ المراقب الذكي (الحالي):
- يحلل المخرجات
- يكتشف نوع المشكلة
- يحل المشكلة بذكاء
- لا يعيد التشغيل إلا بعد الحل

---

## 🔍 المشاكل التي يكتشفها

| المشكلة | الكشف | الحل |
|---------|-------|------|
| **Rate Limit** | `429` أو `Rate limit` | انتظار 60 ثانية ثم إعادة التشغيل |
| **قاعدة بيانات مقفلة** | `SQLITE_BUSY` | انتظار 10 ثواني ثم إعادة التشغيل |
| **انتهاء المهلة** | `ETIMEDOUT` | انتظار 30 ثانية ثم إعادة التشغيل |
| **فشل الاتصال** | `ECONNREFUSED` | انتظار 30 ثانية ثم إعادة التشغيل |
| **نفذت الذاكرة** | `out of memory` | انتظار 20 ثانية ثم إعادة التشغيل |
| **مشكلة المفتاح** | `401` أو `Unauthorized` | ⚠️ تنبيه - يحتاج تدخل يدوي |
| **خطأ SQL** | `syntax error` | ⚠️ تنبيه - يحتاج تدخل يدوي |
| **مورد غير موجود** | `404` | تسجيل وتجاهل |

## 📊 الفحص الذكي

### كل 3 دقائق يفحص:

1. **حالة العملية:**
   - هل نشطة؟
   - هل توقفت؟

2. **تحليل التقدم:**
   - هل تعمل؟
   - هل تعالج بيانات؟
   - ما السرعة؟

3. **فحص قاعدة البيانات:**
   - كم متبقي؟
   - هل اكتمل؟
   - هل هناك خطأ؟

4. **تحليل المخرجات:**
   - هل هناك أخطاء؟
   - ما نوعها؟
   - كيف نحلها؟

---

## 🔧 آلية الحل الذكي

```
اكتشاف المشكلة
    ↓
تحليل النوع
    ↓
اختيار الحل المناسب
    ↓
تطبيق الحل
    ↓
إعادة التشغيل (إذا لزم)
```

### أمثلة:

#### Rate Limit:
```
1. اكتشاف: "429 Too Many Requests"
2. التحليل: تجاوز حد الطلبات
3. الحل: انتظار 60 ثانية
4. الإجراء: إعادة التشغيل بعد الانتظار
```

#### قاعدة بيانات مقفلة:
```
1. اكتشاف: "SQLITE_BUSY"
2. التحليل: قاعدة البيانات مقفلة
3. الحل: انتظار 10 ثواني
4. الإجراء: إعادة التشغيل بعد تحرير القفل
```

#### مشكلة المفتاح:
```
1. اكتشاف: "401 Unauthorized"
2. التحليل: مشكلة في API Key
3. الحل: لا يمكن الحل تلقائياً
4. الإجراء: تنبيه المستخدم
```
