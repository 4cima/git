# 🎯 قواعد قياس جودة البيانات

## ⚠️ قاعدة أساسية - لا تُنسى أبداً

### ❌ خطأ: الاحتكام للبيانات الثابتة
**لا تقيس جودة البيانات بناءً على الـ IDs أو البيانات الثابتة!**

مثال خاطئ:
- عندنا 1,000,000 ID أفلام
- سحبنا 10,000 فيلم
- 9,998 منهم مكتمل
- ❌ **خطأ:** تقول النسبة 0.99% (9,998 / 1,000,000)

### ✅ صحيح: الاحتكام للبيانات المسحوبة فعلياً
**قِس جودة البيانات بناءً على ما تم سحبه فعلياً!**

مثال صحيح:
- عندنا 1,000,000 ID أفلام (بيانات ثابتة)
- سحبنا 10,000 فيلم (بيانات متغيرة)
- 9,998 منهم مكتمل
- ✅ **صحيح:** النسبة 99.98% (9,998 / 10,000)

## 📊 كيفية القياس الصحيح

```sql
-- ❌ خطأ: القياس على كل الـ IDs
SELECT COUNT(*) FROM movies WHERE overview_ar IS NOT NULL

-- ✅ صحيح: القياس على المسحوب فقط
SELECT COUNT(*) FROM movies 
WHERE overview_en IS NOT NULL  -- البيانات المسحوبة
AND overview_ar IS NOT NULL    -- البيانات المترجمة
```

## 🔑 القاعدة الذهبية

**البيانات الثابتة (IDs) ≠ البيانات المسحوبة**

- **البيانات الثابتة:** IDs منزلة من المكتبة مرة واحدة
- **البيانات المتغيرة:** البيانات المسحوبة من TMDB API

**احتكم دائماً للبيانات المتغيرة عند قياس الجودة!**
